# Introduction

This repository documents my journey through the **42 Inception** project.

The purpose of this guide is not only to provide a working solution, but also to explain every technology used throughout the project from the ground up.

Instead of simply following tutorials, we will understand **what each technology is**, **why it exists**, **how it works**, and **how every component communicates with the others**.

By the end of this guide, you will not only know how to build the project—you will understand what happens behind the scenes every time someone opens the website.

---

## What Is Inception?

Inception is a **System Administration** project in the **42 curriculum**.

The goal is to build a complete web infrastructure using Docker.

Instead of installing services directly on the operating system, each service runs inside its own isolated container. These containers communicate through a Docker network while persistent data is stored using Docker volumes.
